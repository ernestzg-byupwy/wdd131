HTML

Role: Structure

History

HTML was invented by Tim Berners-Lee in 1991 while working at CERN, the European physics laboratory in Switzerland. His original goal was simple: create a way for scientists to share documents over a network. The first version supported only 18 elements, mostly for headings, paragraphs, and links.

In 1995, HTML 2.0 became the first formally published standard, followed by HTML 3.2 and HTML 4.01 in the late 1990s as the web grew rapidly in popularity. During this era, browsers like Netscape and Internet Explorer often introduced their own proprietary tags, leading to significant compatibility headaches for developers.

In 2000, the W3C (World Wide Web Consortium) introduced XHTML, a stricter, XML-based reformulation of HTML. It aimed to enforce better coding discipline, but its rigidity made it difficult to adopt widely.

The most significant leap came with HTML5, whose development began around 2004 led by the WHATWG — a group formed by Apple, Mozilla, and Opera in response to the W3C's direction. HTML5 introduced native support for audio, video, the canvas element, semantic elements, and much more. It was officially published in 2014 and is now maintained as a "Living Standard," meaning it is continuously updated rather than released in fixed versions.

Today HTML is the backbone of every webpage on the internet.

Basic Syntax

                    
                        <!DOCTYPE html>
                        <html lang="en">
                         <head>
                              <meta charset="UTF-8">
                              <meta name="viewport" content="width=device-width, initial-scale=1.0">
                              <title>Page Title</title>
                        </head>
                        <body>
                         <h1>Hello, World!</h1>
                        </body>
                        </html>

Common Elements

Element Purpose
<h1> – <h6> Headings (h1 = most important)
<p> Paragraph
<a href=""> Hyperlink
<img src="" alt=""> Image (void element — no closing tag)
<ul> / <ol> / <li> Unordered / ordered list / list item
<div> Generic block container
<span> Generic inline container

Semantic Elements

Semantic elements describe their meaning to the browser and developers alike:

Element Meaning
<header> Introductory content or nav group
<nav> Navigation links
<main> Primary content of the page
<section> Thematic grouping of content
<article> Self-contained content
<aside> Secondary/sidebar content
<footer> Generic inline container

Void Elements (self-closing)

These elements have no content and no closing tag: <br>, <hr>, <img>, <input>, <link>, <meta> <link>, <meta>
