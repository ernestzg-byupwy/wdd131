JavaScript

Role: Behavior/Interaction

History

JavaScript was created by Brendan Eich in just 10 days in May 1995 while he was working at Netscape Communications. The goal was to create a lightweight scripting language that could run directly in the Netscape Navigator browser and make web pages interactive without requiring a server round-trip.

Despite its rushed origin, JavaScript shipped in Netscape Navigator 2.0 in December 1995. Microsoft quickly reverse-engineered it and released their own implementation called JScript in Internet Explorer 3.0 in 1996 — beginning the browser compatibility wars.

To standardize the language, Netscape submitted JavaScript to ECMA International in 1996. The result, published in 1997, was called ECMAScript (ES) — the official name of the language standard. JavaScript remains the most common name in practice.

Progress was slow through the 2000s. ES3 (1999) was widely supported; ES4 was abandoned due to disagreements. ES5 (2009) brought important improvements like strict mode and array methods. ES6 — officially ES2015 — was a landmark release introducing let, const, arrow functions, classes, template literals, modules, and Promises.

Since ES2015, the language has followed an annual release cycle. Today JavaScript runs not only in every browser but also on servers (Node.js), mobile apps, desktop applications, and even embedded systems. It is consistently ranked as the most widely-used programming language in the world.

Basic Syntax

Including JavaScript in HTML 
                    
                        <!-- Recommended: external file with defer -->
                        <script src="scripts/main.js" defer> </script>

The defer attribute ensures the HTML loads completely before the script runs.
